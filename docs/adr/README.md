# Architecture Decision Records

Architecture Decision Records (ADRs) preserve the context, alternatives, and
consequences behind material Prismdrake decisions. An implementation does not
make a Proposed decision Accepted; maintainers approve status changes through
review.

## Status model

- **Proposed:** under active review and not authoritative.
- **Accepted:** approved and authoritative.
- **Rejected:** evaluated and intentionally not selected.
- **Superseded:** replaced by a named newer ADR.
- **Deprecated:** retained for history but no longer recommended.

Every ADR contains a title, status, date, owners, context, decision drivers,
considered options, decision, consequences, validation or evidence, revisit
conditions, and references. Accepted ADRs contain no unresolved placeholder
markers.
