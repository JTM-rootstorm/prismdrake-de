# Prismdrake visual language

Prismdrake uses one semantic design system with two profiles. This PD0 document
and its mockups describe direction and hierarchy, not production artwork.

## Shared identity

Both profiles use restrained prismatic facets, clear state borders, direct
spatial hierarchy, original generic icon geometry, and typography selected from
the user's available generic sans-serif family. They share component code and
required tokens. Profile switching replaces one validated immutable token
generation; it never forks the shell or mixes generations.

Familiarity comes from the traditional panel, launcher, tasks, notification,
and settings relationships. No proprietary asset, logo, sound, font, or exact
interface geometry is a source file for Prismdrake.

## Prismdrake Lustre

Lustre is translucent, layered, and luminous. Fine directional edges and small
facet highlights create a prismatic identity without overwhelming text.
Rounding is moderate and motion is smooth, brief, interruptible, and
non-blocking. Blur is a compositor request only. Every translucent material has
an alpha-tinted or opaque fallback that preserves borders and contrast.

## Prismdrake Forge

Forge is mostly opaque, tactile, crisp, and compact. Strong borders, tighter
geometry, short state transitions, and pronounced surface separation provide a
classic physical quality. Accents are saturated but adjustable. Active,
pressed, focused, and urgent states remain explicit and do not rely on color
alone.

## State vocabulary

Default, hover, focus, pressed, active, urgent, disabled, warning, and error
states require visible semantic tokens and accessible state. Focus is never
represented by color alone. Urgency combines border/shape emphasis with an
accessible label; animation may supplement but never carry the meaning.

## Originality review

Mockups use abstract glyphs and generic application names. Production icons,
wallpapers, sounds, and detailed brand art require provenance and intentional
review. Code-generated geometry remains subject to the same originality rule.
